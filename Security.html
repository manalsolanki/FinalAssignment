Measures Involved

  • X-Frame Options : This type of attack generally means that the attacker can embed this website as iframe to a third party website. Or onclick injection may occur. To prevent this X-Frame-Options HTTP header is included in _headers file.
  • X :XSS-Protection By including this to header, it prevents the site if the page is being attacked.
  • X-Content-Type-Options :

    It will stop a browser from trying to MIME sniff the content type and force to decalred type.

  • Content-Security-Policy :

    This header is effective measure to prevent the site from XSS attacks.It allows to control how to load on a page.

  • X-XSS-Protection :

    This instructs the browser to stop loading when detected Cross site scripting attacks.This controls that how much data is to be sent with each request originated from current website.
